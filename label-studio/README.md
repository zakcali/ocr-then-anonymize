# Label Studio Integration Pipeline

This directory contains utility scripts designed to bridge the gap between the local LLM anonymization pipeline and [Label Studio](https://labelstud.io/), an open-source data labeling tool. 

These scripts convert the LLM's raw markdown outputs into character-level coordinate predictions, merge outputs from multiple models, and align them with human "Gold Standard" annotations for evaluation.

## Directory Structure

```text
label-studio/
├── ls_import_labelling.py            # Converts raw LLM markdown output into Label Studio JSON
├── ls_merge_predictions.py           # Merges multiple model JSONs into a single predictions file
├── ls_merge_gold_predictions.py      # Merges predictions with human annotations (Gold Standard)
├── command to merge json files.txt   # Reference for CLI commands used to merge JSONs
├── golden-standard.json              # Human-annotated Ground Truth exported from Label Studio
├── predictions.json                  # All models' predictions merged into one file
├── golden_merged_predictions.json    # Final file: Human annotations + All Model predictions
└── label_studio_*_predictions.json   # Individual prediction files per model
```

## Workflow & Scripts

### 1. Generating Individual Model Predictions
**Script:** `ls_import_labelling.py`

This script processes the output of a specific model (generated during the `anonymize` stage using the `anonym_pre.txt` XML wrapping prompt). 
- It reads both the original unmodified `.txt` files and the model's `.md` outputs.
- It parses the XML tags (e.g., `<NAME>...</NAME>`) generated by the model.
- It searches for those exact strings in the *original* text to calculate the exact start and end character coordinates.
- It outputs a Label Studio compatible JSON file (e.g., `label_studio_Qwen3.6-35B-A3B-NonThinking_predictions.json`).

*Configuration:* Edit the `MODEL_NAME` and folder paths at the top of the script before running.

### 2. Merging Multiple Models
**Script:** `ls_merge_predictions.py`

When evaluating multiple models simultaneously, it is highly efficient to load them into Label Studio as distinct "prediction" sets on the same task. This script merges multiple individual JSON files into a single `predictions.json` file.

**Usage Example:**
```bash
uv run ls_merge_predictions.py -o predictions.json \
  label_studio_gemma-27b-it_predictions.json \
  label_studio_Qwen3.5-27B-NonThinking_predictions.json \
  label_studio_gpt-oss-120b-Thinking_predictions.json
```
*(See `command to merge json files.txt` for the full 10-model merge command).*

### 3. Creating the Evaluation Dataset
**Script:** `ls_merge_gold_predictions.py`

To calculate metrics or visually compare model performance against human labelers, this script merges the `predictions.json` (from step 2) with a `golden-standard.json` file (exported manually from Label Studio after human annotation).

- It maps the human annotations (`annotations` array) and the model outputs (`predictions` array) to the same tasks based on the file's relative path.
- It outputs `golden_merged_predictions.json`.

**Usage:**
Ensure `predictions.json` and `golden-standard.json` are in the directory, then run:
```bash
python ls_merge_gold_predictions.py
```
The resulting `golden_merged_predictions.json` can be imported back into Label Studio or passed to the `statistics/` folder for automated metric calculation (Precision, Recall, F1, Confusion Matrices).
